# 动物视频工作室 (Animal Video Studio)

## 项目概述
Animal Video Studio 是一个自动化的视频生产流水线，旨在生成自然纪录片风格的短视频。它利用 AI 模型（通过即梦/Jimeng API）来处理整个创作过程：从剧本编写、角色/场景设计到图像和视频生成。

该项目旨在生产高质量、科学准确且具有娱乐性的“科普短视频”，重点关注动物行为和生态位。

### 核心技术
- **运行时:** Node.js (ES Modules)
- **AI 集成:** 即梦 (Jimeng) API，用于图像和视频生成。
- **后端:** 使用原生 `node:http` 构建的自定义 HTTP 服务器 (`server.js`)。
- **前端:** `public/` 目录下的原生 JS/HTML/CSS。
- **数据持久化:** 本地 JSON 配置和版本化的任务文件夹。

---

## 架构与目录结构
- `server.js`: 主入口文件。提供用于任务管理、设置和触发生成流程的 REST API。
- `lib/`:
    - `workflow.js`: 项目的“大脑”。包含动物档案、叙事风格库、受众预设，以及生成剧本、资产列表和镜头列表的逻辑。
    - `jimeng.js`: 即梦 API 的封装，处理图像生成、视频生成和文件下载请求。
- `tasks/`: 存储所有生成的内容。
    - 按 `[动物名称]/versions/[版本 ID]/` 组织。
    - 每个版本文件夹包含：
        - `manifest.json`: 任务的完整状态。
        - `assets.json`, `shots.json`, `voice.json`: 特定的计划数据。
        - `images/`: 生成的资产（角色设计、场景、镜头首帧）。
        - `videos/`: 生成的视频片段。
        - `解说方案.md`, `口播文案.txt`, `剪映导入清单.md`: 可读的输出文件。
- `config/`: 配置文件（如 `settings.json`，存储 API 基础地址和会话令牌）。
- `public/`: 用于监控和控制工作室的 Web 界面。
- `docs/`: 补充文档（如用户体验分析）。

## 快速入门

### 前置条件
- Node.js (推荐 v18+，支持 `fetch` 和 ESM)。
- 正在运行的即梦 API 提供程序（默认指向 `http://127.0.0.1:5100`）。

### 关键命令
- **启动服务器:** `npm start`
- **静态语法检查:** `npm run check` (验证 `server.js`、`lib/` 和 `public/app.js`)
- **控制脚本:** `start.sh`、`stop.sh`、`restart.sh` 用于进程管理。

## 开发与工作流

### 1. 添加新的动物档案
编辑 `lib/workflow.js` 并在 `ANIMAL_PROFILES` 中添加条目。档案包括：
- `habitat` (栖息地), `look` (外貌), `facts` (科学事实)。
- `props` (用于提示词的背景道具)。
- `personality` (性格) 和 `humor` (幽默感)（叙事特征）。
- `styleId` (默认叙事风格)。

### 2. 叙事风格
`lib/workflow.js` 中的 `STYLE_LIBRARY` 定义了 AI 如何编写剧本。当前风格包括：
- `deadpan` (冷面吐槽型)
- `action` (热血闯关型)
- `cozy` (治愈轻喜剧型)
- `mystery` (悬疑揭秘型)

### 3. 生成流水线
1. **方案建议:** 确定动物、风格和受众。
2. **剧本生成:** 构建约 20 句带有科学“钩子”的剧本。
3. **资产规划:** 为“主角”角色和各种场景/道具创建提示词。
4. **镜头规划:** 生成镜头列表，每个镜头对应剧本中的一句话。
5. **制作阶段:**
    - 生成角色和场景的图像。
    - 为每个镜头生成“首帧图”（合成角色 + 场景）。
    - 从首帧图生成视频。

## AI 代理指令指南
在本项目工作时：
- **仅限 ESM:** 始终使用 `import/export` 语法。不要使用 `require`。
- **外科手术式编辑:** 修改 `workflow.js` 时，确保基于模板的剧本生成逻辑与现有模式保持一致。
- **提示词工程:** `workflow.js` 中的提示词使用特定的关键词，如“高清写实动物科普”和“无文字，无水印”，以保持风格一致。在添加新资产时请保留这些词。
- **任务完整性:** 修改任务状态时，务必通过 `saveTask` 更新 `manifest.json`，以保持前后端同步。
- **路径处理:** 处理绝对路径与相对路径时要小心，特别是将资产下载到版本化的 `tasks/` 子目录时。
